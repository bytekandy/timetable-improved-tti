# Timetable Scheduler Test Cases

This document meticulously tracks the logical scenarios verified in the IIIT Dharwad Timetable Scheduler. It aims to extensively document edge-cases relating to Faculty mapping, Room constraints, and student overlap conditions. 

| ID | Detailed Category | In-Depth Scenario Description | Execution Result | Elaborated Findings & Bug Fixes |

| TC01 | Student Cohort Constraints | **Core vs Core Slot Collision:** Two core courses belonging to the exact same branch and semester (e.g., CSE Sem 1). The algorithm must inherently restrict these courses from ever overlapping into the identical time slot across any given day, preserving the integrity of that single student group's schedule. | ✅ Pass | Evaluated correctly. Both courses successfully placed in distinct isolated time slots without colliding. |
| TC02 | Student Cohort Constraints | **Core vs Elective Interaction Drop:** A branch-specific Core course mixed against an Elective course for the same semester/branch. | TC03 | Student Cohort Constraints | **Elective Independence Rule:** Two competing Elective courses offered exclusively to the exact same branch and semester.
